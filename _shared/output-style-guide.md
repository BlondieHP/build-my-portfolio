# 输出风格指南

所有 AI 输出必须遵循此指南，确保跨阶段的视觉一致性。

## 状态图标

| 图标 | 含义 | 使用场景 |
|------|------|---------|
| ✅ | 成功/完成 | 阶段完成、任务完成、验收通过 |
| 🔄 | 进行中 | 当前正在执行的步骤 |
| ⚠️ | 需要注意 | 错误、冲突、需要用户决策 |
| ⚪ | 未开始 | 尚未进入的阶段或步骤 |
| ? | 待确认 | 需要用户输入或确认 |
| 💡 | 提示/洞察 | 补充说明、最佳实践提示 |
| 📝 | 笔记 | 自动捕获到 NOTES.md 时 |

## 阶段标题格式

```
## 🔄 Phase X: [阶段名] — [当前步骤]
```

## 确认词列表

以下词汇视为用户确认，AI 可继续推进：
- "继续"、"好的"、"OK"、"懂了"、"理解"、"明白了"、"可以"、"行"、"开始"、"执行"、"下一步"

不在列表中的回复 → 视为提问或反馈，不推进流程。

## 输出密度

- 概念解释：3-5 句，超过则分段
- 代码展示：只展示关键片段，不超过 20 行
- 选项列表：每次给 2-4 个选项，不更多
- 交接消息：2-3 句话，不含赘述

## 进度展示

涉及多个步骤时，使用进度概览：

```
里程碑进度：
████████░░░░░░░░  2/4 (50%)
✅ 里程碑 1: 环境搭建
🔄 里程碑 2: 核心功能 — 进行中
⚪ 里程碑 3: 扩展功能
⚪ 里程碑 4: 测试与文档
```

## 代码审查格式

```
审查：
✅ [通过的方面]
⚠️ [需要注意的方面]
💡 [改进建议]
```
