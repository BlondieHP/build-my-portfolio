# Phase 6: 反思 (REFLECT)

**目标**：内化知识，更新学习记录和作品集，决定下一步方向。

## 前置条件
- BUILD 阶段已完成（至少完成了一个项目）
- 或者用户在任何时候想停下来回顾

## 流程

### Step 1: 回顾收获

**先读取 `.project-learning/NOTES.md`**，获取 BUILD 阶段自动捕获的笔记作为回顾原材料。

然后引导用户回答：

1. **知识层面**：学到了哪 3 个最重要的概念？
2. **技能层面**：现在能做什么之前做不到的事？
3. **困惑层面**：还有什么没搞懂的？还想深入什么？

AI 应对照 NOTES.md 补充用户可能遗漏的收获：
- "你的笔记里还记了 [X 概念]，那个其实也很重要"
- "你用的 [Y 方法] 在笔记里出现了 3 次，这是一个核心模式"
- 把 NOTES.md 中"待搞清楚"的条目拿出来，看用户是否已解决

### Step 2: 更新学习日志
在 `.project-learning/JOURNAL.md` 中追加一条记录：

```markdown
## [日期] — [项目名称]

### 完成的项目
- [项目名称]：[GitHub URL]（如果有的话）

### 核心收获
1. [收获 1]
2. [收获 2]
3. [收获 3]

### 仍然困惑的
- [困惑点]

### 时间投入
- 总计约 X 小时

### 下一步想法
- [下一步想做什么]
```

### Step 3: 更新作品集
更新 `.project-learning/PORTFOLIO.md`：
- 将新项目加入项目列表（时间倒序）
- 更新统计数据（总项目数、技术栈、学习时长）
- 更新技能图谱

### Step 4: 决定下一步
给用户 3 个选项：

**A. 继续当前方向**
- 进入下一个里程碑（回到 SPEC 阶段）
- 适合：当前方向还有更多要学，且用户有动力

**B. 在当前方向深耕**
- 把一个项目做得更完善（加测试、加文档、优化性能、写博客）
- 适合：当前项目还有发挥空间，用户想做出一个"精品"

**C. 开启新方向**
- 回到 DIRECT 阶段，探索新的学习方向
- 适合：当前方向学够了，想扩展技能宽度

**D. 休息/消化**
- 不做新的事情，给知识沉淀的时间
- 推荐阅读 RESOURCES.md 中的补充资源
- 建议一个"知识回顾"的日期

### Step 5: 记录决定
在 `.project-learning/JOURNAL.md` 末尾追加下一步决定。

### Step 6: 结束
> 这次学习 session 的收获已经记录好了。你的作品集现在有 X 个项目，涉及 Y 个技术栈。
>
> 你选择的下一个方向是：[选项 A/B/C/D]
>
> 随时回来继续你的学习旅程。下次你只需要说"继续学习"或"开始学习 [新方向]"就行。

## 注意事项
- **反思不是走过场**：这是知识从短期记忆转为长期记忆的关键环节
- **庆祝小胜利**：每完成一个项目都是进步，不要急着赶下一个
- **困惑是正常的**：记录下困惑点本身就是学习——之后回来解决它
- **作品集是给自己的**：不是给别人看的简历，是你自己成长的轨迹
