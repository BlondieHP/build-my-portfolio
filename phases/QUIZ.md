# 跨阶段测验 (QUIZ)

**目标**：在里程碑边界验证学习效果，确保知识内化后再进入下一阶段。

## 触发时机

测验在以下时机触发（AI 应主动检测而非等用户要求）：

1. **里程碑完成时**：BUILD 或 REMIX 阶段中，用户完成了 ROADMAP.md 中一个里程碑的所有任务
2. **用户主动要求**："考我一下"、"测验"、"测测我"
3. **知识点密集模块后**：如果一个里程碑包含 5+ 个新概念，AI 可在中间点插入一次短测验

**不要触发的情况**：
- 用户刚完成一个很短的步骤（不到 3 个新知识点）
- 用户表示疲惫或时间不足（"今天到此为止"）
- REFLECT 阶段（反思本身已有回顾功能）

## 流程

### Step 1: 生成题目
基于当前里程碑的学习目标和 NOTES.md 中记录的收获，AI 生成 **3-5 道题**：

**题型分配**：
- 概念题 × 2：检验是否理解核心概念（"为什么 X 要这样设计？"）
- 实践题 × 1-2：检验是否能做（"如果要实现 Y，你会怎么改？"）
- 陷阱题 × 1（可选）：常见误区（"以下哪个做法是错误的？"）

**出题原则**：
- 只用本里程碑覆盖的知识点
- 不给任何答案线索（选项长度必须一致）
- 题干简洁，不超过 2 句话
- 优先选择题和判断题，减少开放式提问（不给用户负担）

### Step 2: 逐题交互
一次只出一道题。用户回答后：

| 结果 | AI 行为 |
|------|--------|
| ✅ 正确 | 简短确认，"正确。下一题。" |
| ❌ 错误 | 不直接给答案，给一个通向正确答案的提示。如果用户仍然答错，给出解释并追加到 NOTES.md 的"待搞清楚" |
| ? 不确定 | 接受不确定，解释答案后追加到 NOTES.md |

### Step 3: 结果总结
全部完成后给出总结：

```
测验结果：3/4 ✅

✅ Rust 所有权规则 — 理解正确
✅ Cargo 包管理 — 理解正确
✅ Result 错误处理 — 理解正确
⚠️ 生命周期标注 — 需要复习（已记入笔记）

建议：[如果有低于 60% 的，建议回炉该概念后再进入下一里程碑]
```

### Step 4: 记录
将测验结果追加到 NOTES.md 的对应日期条目下：

```markdown
### 测验：[里程碑名称] — 3/4 ✅
- ✅ 所有权规则
- ✅ Cargo 包管理
- ✅ Result 错误处理
- ⚠️ 生命周期标注（待复习）
```

### Step 5: 返回主流程
测验结束后自动返回 BUILD 或 REMIX 的当前步骤，无需用户说"继续"。

## 跳过机制
用户可以跳过一次测验（"跳过测验"），AI 不追问原因。但如果是连续跳过 3 次以上，AI 善意提醒：
> "你已经跳过了 3 次测验。测验帮确认你真的掌握了——想现在试一下吗？"

## 题目范例

```
Q1: 在 Rust 中，以下哪项关于所有权（ownership）的描述是正确的？
A. 一个值可以同时有多个可变引用
B. 一个值在同一时间只能有一个可变引用
C. 所有权规则只在多线程环境下生效
D. 获取值的所有权会自动创建一份副本

（4 个选项长度一致，不给线索）
```
