# ROADMAP.md 格式规范

`ROADMAP.md` 将收敛后的资源编排为有序、可执行的里程碑路径。

## 模板

```markdown
# 学习路线图

## 总览

- **里程碑数量**：3-5 个
- **预计总时间**：X 小时 / X 周
- **每个里程碑产出**：一个可验证的 mini-project 或检查点

---

## 里程碑 1：[名称] — 预计 X 小时

### 学习目标
<!-- 这个阶段要掌握的具体知识点 -->

### 学习材料
<!-- 从 RESOURCES.md 中选取，标注具体章节 -->
- [资源名称] 第 X-Y 章

### 动手实践
<!-- 一个 1-3 小时能完成的小练习 -->
- **Mini-project**：[项目名称] — [一句话描述]
- **验收标准**：[项目能跑起来 / 通过测试 / 实现某个功能]

### 完成标志
- [ ] 能回答：[关键问题]
- [ ] 能做出：[具体产出]

---

## 里程碑 2：[名称]
...
```

## 编排原则

1. **依赖驱动排序**：A 是 B 的前置知识 → A 排在 B 前面。
2. **每个里程碑有产出**：读完书不算完成，做出东西才算。
3. **难度螺旋上升**：每个里程碑在前一个基础上加 20-30% 新内容。
4. **时间粒度**：每个里程碑控制在 2-6 小时可完成，给用户明确的"下一步"感。
5. **与项目解耦**：ROADMAP 是学习路径，SPEC 是具体一个项目的规约。ROADMAP 中的 mini-project 可以发展为后续的大项目。
