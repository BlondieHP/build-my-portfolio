# MISSION.md 格式规范

`MISSION.md` 是学习旅程的北极星。所有后续阶段都锚定于此。

## 模板

```markdown
# 学习使命

## 我想学什么
<!-- 一句话描述学习方向，例如："用 Rust 写命令行工具" -->

## 为什么想学
<!-- 深层动机。不是为了简历好看——你想用它创造什么？解决什么问题？ -->

## 成功标准
<!-- 3-5 条可验证的标准。完成了这些，就算"学会了" -->
- [ ] 标准 1：例如"能独立写出一个带参数解析的 CLI 工具"
- [ ] 标准 2：例如"理解 Rust 所有权模型，能解释给同事听"
- [ ] 标准 3：例如"项目代码推到 GitHub，README 清晰可读"

## 约束条件
<!-- 真实约束，帮助收敛资源范围 -->
- 时间：每周 X 小时，预计 X 周
- 前置知识：已掌握 XXX
- 工具限制：只用免费/开源工具？需要特定平台？

## 启动日期
<!-- YYYY-MM-DD -->

## 修改记录
<!-- 使命可能演进，记录变化时间线 -->
- YYYY-MM-DD：初始版本
```

## 原则

1. **为什么 > 学什么**：动机是坚持的燃料。如果用户写不出"为什么"，先在这一步停下来深挖。
2. **成功标准必须可验证**：不是"理解 Rust"，而是"写出一个能跑的 Rust CLI 工具"。
3. **约束条件要真实**：一周只有 2 小时的学习者和整天都能学的人，路径完全不同。
4. **允许演进**：学到一半发现方向需要调整？更新 MISSION.md 并留下修改记录。
