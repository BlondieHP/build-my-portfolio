# RESOURCES.md 格式规范

`RESOURCES.md` 是经过 AI 筛选和用户确认后的最小必要资源清单。

## 核心约束

**资源总数严格控制在 5-10 个。** 如果超过 10 个，必须进一步收敛。用户不需要知道所有资源，只需要知道最好的那几个。

## 模板

```markdown
# 学习资源清单

## 核心资源（必读，按推荐顺序）

### 1. [资源名称](URL)
- **类型**：官方文档 / 书籍 / 视频课程 / 交互式教程 / 博客文章
- **为什么选它**：一句话说明为什么它是这个阶段最好的选择
- **覆盖范围**：涵盖哪些知识点
- **预计时间**：X 小时
- **难度**：入门 / 进阶 / 深入
- **关键章节**：如果资源很长，标注最值得看的部分

### 2. [资源名称](URL)
...

## 补充资源（可选，加深理解）

<!-- 最多 3 个，给有余力的学习者 -->
...

## 不推荐但常见的资源

<!-- 列出被过滤掉的知名资源，解释为什么没入选 -->
- [资源名称]：原因（太旧 / 太深 / 偏离目标 / 英文门槛 / 信息密度低）
```

## 筛选原则

1. **官方文档优先**：如果官方文档质量高，它就是第一选择。
2. **交互式 > 被动式**：能动手的教程优于纯阅读的材料。
3. **一个概念一个资源**：不要给同一个知识点列 3 个资源，选最好的那个。
4. **解释为什么过滤**："不推荐但常见"列表帮助用户建立信任——让他知道你做了功课。
5. **尊重语言偏好**：中文用户优先中文资源，但不要为了语言牺牲质量。
