# SPEC.md 格式规范

`SPEC.md` 是单个项目的规约文件，将当前阶段的理论知识映射为具体的 demo 项目。

## 模板

```markdown
# 项目规约：[项目名称]

## 一句话描述
<!-- 这个项目是什么，谁会用，解决什么问题 -->

## 学习映射
<!-- 这个项目练习哪些知识点 -->
| 知识点 | 项目中体现在哪里 |
|--------|-----------------|
| [概念 A] | [在项目的哪个部分用到] |
| [概念 B] | [在项目的哪个部分用到] |

## 技术栈
- 语言：
- 框架/库：
- 工具：

## 功能需求
### 核心功能（必须完成）
1. [功能 1]
2. [功能 2]
3. [功能 3]

### 扩展功能（有余力再做）
1. [功能 4]

## 项目结构
```
<project-slug>/
├── README.md
├── src/
│   └── ...
├── tests/
│   └── ...
└── ...
```

## 验收标准
<!-- 做完这些才算"完成" -->
- [ ] 项目可以独立运行（clone → install → run 一条命令）
- [ ] README.md 说明项目是什么、怎么跑
- [ ] 核心功能全部实现
- [ ] 代码有基本注释（解释关键决策，而非显而易见的操作）
