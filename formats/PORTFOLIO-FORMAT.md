# PORTFOLIO.md 格式规范

`PORTFOLIO.md` 是所有已完成项目的索引，类似 GitHub 个人主页的作品集。

## 模板

```markdown
# 我的学习作品集

## 统计
- 总项目数：X
- 涉及技术栈：Rust, TypeScript, Python, ...
- 学习时长：累计 X 小时

---

## 项目列表（按时间倒序）

### [项目名称] ⭐（推荐展示）
- **仓库**：[GitHub URL]
- **完成日期**：YYYY-MM-DD
- **一句话描述**：用 X 技术做了 Y，解决了 Z 问题
- **学会的技能**：技能 A, 技能 B, 技能 C
- **技术栈**：Rust, Clap, Serde
- **亮点**：
  - 实现了 XXX
  - 测试覆盖率 XX%
- **相关博客/笔记**：[如果有的话]
- **截图/Demo**：[如果有的话]

---

## 技能图谱
<!-- 按领域分组，展示掌握的技能 -->

### 后端开发
- Rust：项目 [A](link), [B](link)
- Python：项目 [C](link)

### 前端开发
- React：项目 [D](link)

---

## 下一步
<!-- 当前正在学什么，下一个想攻克的方向 -->
- 进行中：[当前项目/方向]
- 计划中：[下一个想学的]
```

## 更新原则

1. **每完成一个项目立即更新**：延迟 = 遗忘。
2. **统计自动汇总**：AI 应在 REFLECT 阶段自动更新统计数据。
3. **项目按时间倒序**：最新的在最前面，展示成长轨迹。
4. **亮点 > 功能列表**：不要说"实现了 CRUD"，要说"实现了支持模糊搜索的全文检索引擎"。
