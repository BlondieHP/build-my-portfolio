# NOTES.md 格式规范

`NOTES.md` 是学习过程中的实时笔记，由 AI 在学习过程中**自动捕获**关键收获，而非等到反思阶段才记录。

## 与 JOURNAL.md 的区别

| | NOTES.md | JOURNAL.md |
|------|----------|------------|
| **更新时机** | 实时，每次 BUILD session | 阶段性，每个项目完成后 |
| **粒度** | 细——每条命令、每个概念 | 粗——核心收获、困惑点 |
| **谁写** | AI 自动捕获 + 用户确认 | 用户主导，AI 辅助 |
| **用途** | 过程中查阅、复习 | 阶段性总结、作品集配套 |

## 模板

```markdown
# 学习笔记

## 2026-XX-XX — [当前学习主题]

### 理解的概念
<!-- AI 在解释概念后自动捕获 -->
- **[概念名]**：[一句话总结]
  - 为什么重要：[一句话]
  - 相关命令：[如果有]

### 学到的命令
<!-- 新掌握的 CLI 操作 -->
| 命令 | 作用 | 示例 |
|------|------|------|
| `git init` | 初始化仓库，创建 .git | `git init` |
| `git status` | 查看文件状态 | `git status` |

### 踩过的坑
<!-- 出错 → 原因 → 怎么解决的 -->
- **问题**：[现象]
  - **原因**：[根因]
  - **解决**：[方法]

### AI 讲解摘要
<!-- AI 解释过的关键概念，简短记录 -->
- [概念]：[一句话解释]

### 待搞清楚
<!-- 暂时跳过但想回来弄懂的 -->
- [ ] [问题]
```

## 自动捕获规则

在 BUILD 阶段，AI 应在以下时机**主动捕获**笔记：

1. **解释新概念后** → 自动追加一条"理解的概念"
2. **用户学会新命令后** → 追加到"学到的命令"表格
3. **用户遇到错误并解决后** → 追加到"踩过的坑"
4. **每个 BUILD session 结束前** → 追加当天时间戳和总结

AI 捕获后应简短确认：
> "我把这个概念记到你的笔记里了：[条目摘要]"

用户可以说"不用记"跳过。

## 格式原则

1. **简洁**：每条笔记 1-2 行，能扫读
2. **结构化**：用表格 > 列表 > 段落
3. **可搜索**：关键词和命令名称清晰
4. **不重复**：同一个概念只记一次（除非有新的理解）
